# Automated Design Deployment

윈도우 기반 프로젝트 파일 배포/관리 자동화 도구

---

## 주요 기능

- **서버(사설/로컬) 접속 및 관리**
  - IP:포트 입력 후 접속/끊기
  - 접속 정보 저장/삭제/자동완성
- **디렉토리/파일 탐색기**
  - 드라이브/폴더/파일 트리 탐색
  - 상위 폴더 이동, 경로 하이라이트, 폴더-파일 구분
- **즐겨찾기 관리**
  - 서버별 즐겨찾기 추가/삭제/이동
  - app-data.json에 통합 저장
- **파일 첨부/변경/업로드**
  - 파일 첨부 후 서버로 전송(_sned 네이밍)
  - 이미지/텍스트 구분, 용량제한 없음
  - 업로드 후 경로 새로고침, 중복시 덮어쓰기 확인
- **에디터/비교(diff)**
  - 파일 선택 시 에디터 1개 표시
  - 첨부파일은 전송 전 미리보기 없이 알림창
  - 변경점 비교(diff), 동일 줄 미표시, 하이라이트
- **로그/콘솔/알림**
  - 네트워크/에러/상태 로그, 하단 패널, 최소화/숨김
- **네이티브 메뉴/시스템 메뉴**
  - Electron 네이티브 메뉴(File/Edit/접속/끊기)
- **데이터 통합 저장**
  - app-data.json에 서버/즐겨찾기 등 모든 데이터 저장
- **빌드/배포/설치형**
  - electron-builder, NSIS로 설치형/포터블 빌드

---

## 설치 및 실행

1. **의존성 설치**
   ```
   npm install
   ```
2. **서버 실행**
   ```
   npm run file-server
   ```
3. **프로그램 실행**
   ```
   npm start
   ```
4. **포트 충돌 시 종료**
   ```
   npm run kill-4000
   ```
5. **설치형/포터블 빌드**
   ```
   npm run dist
   # 또는
   npm run dist:installer
   ```

---

## 파일 구조

- src/func/ : 기능별 모듈 (파일/서버/즐겨찾기 등)
- src/ui/   : UI 렌더링 모듈
- src/infra/: 서버, electron bridge 등 인프라
- public/   : 프론트엔드 리소스 (index.html, renderer.js 등)
- scripts/  : 유틸 스크립트 (포트 종료 등)
- app-data.json : 서버/즐겨찾기 등 통합 데이터

## 참고

- 모든 데이터는 app-data.json에 통합 저장
- 서버는 Node.js 기반, 클라이언트는 Electron 기반
- 설치형(Setup.exe) 또는 포터블로 배포 가능
- 자세한 진행상황은 진행사항.md 참고
